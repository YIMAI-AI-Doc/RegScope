# RegScope 平台设计方案

日期：2026-03-26

## 1. 项目概述

`RegScope` 是一个面向中文用户的全球医药监管情报平台，核心目标是将全球医药法规政策、官方动态、行业解读和相关讨论整合到同一个产品中，帮助用户以更低成本完成以下任务：

- 快速了解全球主要监管机构的最新政策变化
- 按国家、机构、领域、小领域进行持续关注
- 在高价值问题上沉淀结论、证据和争议点，而不是被大量回复淹没

网站定位不是普通资讯站，也不是普通社区论坛，而是一个以监管情报为核心、以结构化订阅为抓手、以结论型问答为沉淀机制的专业平台。

## 2. 已确认设计前提

- 首页主轴采用 `Hybrid` 模式
- 网站界面首发仅提供中文，不提供英文界面
- 平台定位为 `情报 + 社区并重`
- 首发核心用户为 `泛医药行业从业者`
- 访问方式采用 `游客可浏览主要内容，登录后才能关注、评论、订阅`
- 首发最强能力优先级为：
  - 全球法规政策抓取与更新速度
  - 领域订阅与个性化关注

## 3. 产品目标

### 3.1 核心目标

1. 建立全球医药监管情报的高频更新入口
2. 让用户能够按结构化主题订阅自己真正关心的内容
3. 让讨论区能够提炼重点而不是制造噪音

### 3.2 非目标

- 首发阶段不追求覆盖所有国家和所有机构
- 首发阶段不做重社交化社区玩法
- 首发阶段不追求复杂会员权限体系
- 首发阶段不以英文、多语言、国际化为优先事项

## 4. 目标用户

首发阶段面向泛医药行业从业者，包括但不限于：

- 注册法规人员
- 药政与合规团队
- 医药研发与临床从业者
- CMC、生产、质量人员
- 咨询与情报分析人员
- 对全球药政趋势有持续关注需求的行业用户

因此产品表达需要遵循一个原则：`入口易懂，进入后足够专业`。

## 5. 产品策略

建议采用以下产品策略：

### 5.1 情报主轴

平台的第一感知应该是“这里能高效看到全球监管更新”，而不是“这里是一个讨论社区”。

### 5.2 订阅增强

用户可以通过国家、机构、大领域、小领域建立个性化关注路径，让产品具备持续使用价值。

### 5.3 结论型社区

讨论区不做传统论坛，而做问题驱动、证据分层、结论沉淀的问答区，服务于信息提炼。

## 6. 站点结构

主导航建议如下：

- `首页`
- `情报快讯`
- `领域订阅`
- `官方来源`
- `国家地区`
- `讨论问答`
- `我的关注`

### 6.1 首页

混合总览页，承担平台价值展示、快速浏览和关注引导功能。

### 6.2 情报快讯

按时间和条件浏览全球监管更新，强调速度、连续性和筛选效率。

### 6.3 领域订阅

按大领域和小领域建立结构化订阅入口，是平台长期留存的关键页面。

### 6.4 官方来源

展示重要官方机构及其最新动态，作为结构化来源入口。

### 6.5 国家地区

从市场和监管辖区角度浏览情报，服务于跨国观察和地区比较。

### 6.6 讨论问答

围绕明确问题组织讨论，并将重点提炼为结论与证据。

### 6.7 我的关注

登录用户管理关注对象和提醒方式的个人页。

## 7. 首页信息架构

首页推荐顺序如下：

1. `今日监管预警`
2. `我的关注中心`
3. `情报资讯大卡片流`
4. `官方来源 / 小领域 / 国家入口小卡片矩阵`
5. `精选讨论摘要`
6. `趋势榜单`

### 7.1 今日监管预警

展示最高优先级的政策、指南、征求意见稿、药物警戒信息或高影响动态。

推荐字段：

- 标题
- 国家 / 地区
- 机构
- 大领域 / 小领域
- 紧急程度
- 3 行摘要

### 7.2 我的关注中心

登录后显示用户已关注的国家、机构、大领域和小领域，以及站内提醒入口。

### 7.3 情报资讯大卡片流

首页主体内容区，承担主要浏览任务。

### 7.4 小卡片矩阵

提供结构化入口，帮助用户在“来源、国家、领域”之间高效切换。

### 7.5 精选讨论摘要

不直接展示大量帖子，而是展示若干高价值问题的当前结论摘要。

### 7.6 趋势榜单

展示今日热度较高的国家、机构、领域和问题，帮助用户发现重点。

## 8. 内容模型与卡片设计

平台建议使用两套核心内容单元。

### 8.1 资讯大卡片

用于承载政策更新、新闻、解读和快讯，风格参考内容平台卡片流，但信息表达更专业。

建议字段：

- 标题
- 来源机构
- 国家 / 地区
- 大领域 / 小领域
- 发布时间
- 2 到 3 行摘要
- 原文入口
- 收藏 / 关注 / 评论数

适用场景：

- 首页主内容流
- 情报快讯列表页
- 领域页中的最新内容区
- 来源页中的发布记录区

### 8.2 小卡片矩阵

用于承载官方机构、国家入口、细分领域、热门订阅等结构化入口。

建议字段：

- 图标 / 徽标
- 名称
- 一句说明
- 关注按钮

适用场景：

- 首页右侧矩阵
- 官方来源页
- 国家地区页
- 领域订阅页

## 9. 领域树设计

建议采用 `双层混合拆分`。

### 9.1 拆分原则

- 大领域按监管流程拆分
- 小领域按具体议题或热点主题拆分

这样既能保持专业精度，也能让泛医药用户快速理解入口。

### 9.2 推荐一级领域

建议首发阶段控制在 8 个一级领域：

- 临床试验
- CMC 与生产
- 注册申报
- 标签与说明书
- 上市后管理
- 药物警戒
- 数字化与 AI 监管
- 国际协调与指南

### 9.3 小领域示例

- `CMC 与生产 -> 稳定性、连续生产、杂质控制`
- `临床试验 -> 方案设计、去中心化临床、真实世界证据`
- `数字化与 AI 监管 -> AI 辅助审评、数据治理、数字工具监管`
- `上市后管理 -> 变更申报、再评价、风险管理`

## 10. 页面模型

### 10.1 首页

定位为“全球医药监管情报总控台”，强调总览、发现和订阅。

### 10.2 情报快讯列表页

采用大卡片流加多维筛选，建议支持以下筛选项：

- 国家 / 地区
- 官方机构
- 大领域
- 小领域
- 内容类型
- 时间范围

### 10.3 资讯详情页

建议结构：

- 标题与标签
- 3 行导读摘要
- 正文或提炼内容
- 原文链接
- 相关领域
- 相关来源
- 相关讨论

该页不仅是阅读页，也是连接订阅体系和讨论体系的内容节点。

### 10.4 领域页

建议包含：

- 领域简介
- 订阅按钮
- 热门小领域
- 最新资讯
- 相关讨论
- 推荐来源

### 10.5 来源页

建议包含：

- 机构介绍
- 关注按钮
- 最新发布
- 高频领域
- 相关国家 / 地区
- 关联讨论

### 10.6 讨论列表页

每条问题卡建议展示：

- 问题标题
- 领域标签
- 当前状态
- 当前结论一句话
- 证据数量
- 最后更新时间

目标是在列表页就帮助用户判断问题价值。

### 10.7 讨论详情页

建议固定结构：

- 问题定义
- 当前结论
- 证据区
- 争议点
- 回复区

其中结论和证据优先于楼层展开。

## 11. 讨论区设计

讨论区建议采用 `结论型问答板`，而不是普通论坛或纯时间流帖子系统。

### 11.1 设计目标

- 让用户快速看懂问题当前结论
- 让证据和观点分层展示
- 让高价值讨论能够持续更新
- 避免长回复淹没重点

### 11.2 问题页五段式结构

#### A. 问题定义区

说明问题边界、适用国家 / 机构、领域标签和背景信息，避免讨论发散。

#### B. 当前结论区

默认置顶，建议固定包含：

- 当前阶段结论
- 适用前提
- 是否已有官方依据
- 最后更新时间

#### C. 证据区

建议按强度分层展示：

- 官方原文
- 权威解读
- 社区经验

#### D. 争议点区

展示：

- 已形成共识的点
- 仍存在争议的点
- 仍缺失的关键证据

#### E. 回复区

回复默认不按纯时间排序，建议优先展示：

- 带证据的回复
- 高认可度回复
- 最新补充

## 12. 防止信息淹没的机制

建议至少实现以下 4 个机制。

### 12.1 问题状态机制

每个问题必须有明确状态：

- 待解答
- 讨论中
- 已形成阶段性结论
- 高争议

### 12.2 回答标签机制

每条回答建议支持以下标签：

- 官方依据
- 行业解读
- 经验判断
- 待核实

### 12.3 自动讨论摘要

当回复较多时，自动生成：

- 3 行摘要
- 核心分歧
- 引用最多的证据链接

### 12.4 结论持续更新

结论不是一次性固定内容，而是随着新政策、新指南或新口径不断修订的动态内容。

## 13. 数据与内容标准化

为了支撑后续筛选、订阅、推荐、讨论关联，所有抓取到的内容建议在入库时标准化为统一结构。

建议标准字段包括：

- 标题
- 摘要
- 原始链接
- 国家 / 地区
- 来源机构
- 内容类型
- 大领域
- 小领域
- 发布时间
- 抓取时间

## 14. 访问与权限策略

首发阶段建议采用 `内容开放浏览，登录后互动与订阅` 的模式。

### 14.1 游客能力

- 浏览首页
- 浏览情报快讯
- 浏览来源页和领域页
- 查看讨论内容

### 14.2 登录用户能力

- 关注国家 / 机构 / 领域
- 评论和回答
- 收藏内容
- 管理我的关注
- 接收站内提醒

## 15. 首发版功能边界

### 15.1 第一优先级

- 全球情报聚合，支持 `RSS / Atom / JSON`
- 内容标准化入库
- 领域订阅
- 首页、列表页、详情页的基础浏览体验

### 15.2 第二优先级

- 官方来源页
- 讨论问答板基础能力
- 讨论与资讯关联

### 15.3 第三优先级

- 自动摘要与争议提取
- 个性化推荐
- 邮件日报 / 周报

## 16. 迭代顺序

### 阶段一

先打通抓取、清洗、分类、首页流、详情页和基础筛选。

目标：让用户相信这里能高效看全球监管更新。

### 阶段二

补齐领域订阅、来源页、国家页和我的关注。

目标：让用户形成持续使用习惯。

### 阶段三

上线结论型讨论区。

目标：让平台从资讯浏览工具升级为情报和判断沉淀平台。

### 阶段四

增强自动摘要、争议识别、推送和推荐。

目标：将平台进一步升级为高效率的监管情报系统。

## 17. 风险与约束

### 17.1 内容源质量风险

如果来源覆盖面过大但质量不稳定，会削弱平台可信度。首发应优先覆盖高价值来源。

### 17.2 分类准确度风险

如果国家、机构、领域、小领域映射不稳定，会直接影响订阅体验和筛选准确性。

### 17.3 社区噪音风险

如果讨论区没有证据标签和结论摘要，社区会快速退化成普通论坛。

### 17.4 首发范围失控风险

如果首发尝试同时覆盖太多国家、太多玩法和复杂会员能力，交付质量会显著下降。

## 18. 结论

`RegScope` 的最佳首发形态是一个中文界面的全球医药监管情报平台：

- 首页采用 `Hybrid` 结构
- 内容浏览以资讯大卡片流为核心
- 结构化入口以官方来源、国家和小领域小卡片为核心
- 订阅体系采用“大领域按流程、小领域按主题”的混合结构
- 讨论区采用结论型问答板，重点沉淀结论、证据和争议
- 首发优先把全球情报抓取与订阅能力做强，再逐步增强讨论提炼和智能摘要
